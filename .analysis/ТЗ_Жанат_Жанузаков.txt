1. Общая информация
2. Цель к пятнице
Поднять базовый Node.js/TypeScript backend с работающими webhook-эндпоинтами для Telegram и WhatsApp, а также REST API для вызова translation-модуля Даниэла. Бэкенд должен принимать сообщения от ботов, обрабатывать их и возвращать перевод.
3. Задачи
4. Технические требования
• Node.js 20+, TypeScript, Express/Fastify
• Архитектура: роуты → контроллеры → сервисы
• Вебхуки должны отвечать за < 1 секунду (async обработка)
• POST /api/translate: { text, targetLang } → { translatedText, detectedLang }
• Секреты только через .env, никаких хардкода ключей в коде
5. Критерии приёмки (Definition of Done)
✓  Сервер запускается локально командой npm run dev
✓  Telegram webhook принимает тестовые запросы (200 OK)
✓  WhatsApp webhook принимает тестовые запросы (200 OK)
✓  /api/translate возвращает перевод через модуль Даниэла
✓  Ернан может подключить своего бота к этим эндпоинтам
6. Синхронизация
До среды, 10 июня — эндпоинты задеплоены локально, Ернан может делать запросы.
До четверга, 11 июня — /api/translate проверен с модулем Даниэла.
Пятница, 12 июня — совместное тестирование полного флоу: бот → backend → перевод.
